# GDPR Data Breach Liability and Security Obligations

## 1. Article 32 Obligations
Under GDPR Article 32, cloud data processors have strict security obligations. They must implement appropriate technical and organizational measures to ensure a level of security appropriate to the risk. This includes pseudonymisation and encryption of personal data, the ability to ensure ongoing confidentiality, integrity, availability, and resilience of processing systems, and a process for regularly testing these measures.

## 2. Controller vs Processor Liability
Under data processing agreements, liability is heavily scrutinized. While the Data Controller bears the primary responsibility for the lawful basis of processing, the Data Processor is strictly liable for any data breaches resulting from a failure to adhere to the Article 32 security obligations or for acting outside the controller's documented instructions.

## 3. Data Breach Notification (Article 33 & 34)
In the event of a personal data breach, the processor must notify the controller without undue delay after becoming aware of it. The controller then has 72 hours to notify the relevant supervisory authority, unless the breach is unlikely to result in a risk to the rights and freedoms of natural persons.

## 4. Contractual Indemnities
Robust Data Processing Agreements (DPAs) will typically include uncapped indemnity clauses requiring the cloud processor to fully compensate the controller for any regulatory fines or third-party damages stemming from the processor's security failures.
